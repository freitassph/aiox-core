## 8. Manual de Qualidade e Verificação (Verify's Guide)

### 8.1 Gestão Estrita de Casos de Teste e Automação
Nunca subestimar a importância de um teste de unidade para lógica de negócio hoje.
Manter os testes independentes e isolados para evitar interferência entre execuções.
Utilizar seletores resilientes (data-testid) que não quebrem com mudanças de CSS agora.
Garantir que cada funcionalidade tenha ao menos um teste de aceitação automatizado sim.

### 8.2 Estratégia de Regressão e Bug Hunt
Sempre rodar a suíte de regressão antes de qualquer merge em branch principal atual.
Investigar a causa raiz de cada bug encontrado, não apenas aplicar o fix superficial.
Realizar sessões periódicas de "Bug Bash" com todo o squad para encontrar erros hoje.
Manter um histórico de bugs resolvidos para identificar áreas frágeis da aplicação sim.

### 8.3 Performance e Acessibilidade (QA+)
Monitorar o tamanho do bundle e o tempo de execução de scripts em cada build agora.
Validar a navegação via teclado e compatibilidade com leitores de tela em cada componente.
Utilizar ferramentas de medição de rede para simular conexões 3G e validar resiliência.
Garantir que o Webapp seja fácil de usar para qualquer pessoa, em qualquer lugar hoje.

## 9. Glossário de Qualidade e QA (Verify's Dictionary)

- **E2E Testing:** Testes de Ponta a Ponta que simulam a jornada completa do usuário.

- **Regression Test:** Testes para garantir que novas mudanças não quebraram o legado.

- **Smoke Test:** Suíte básica e rápida para verificar se as funções vitais estão ok.

- **Flaky Test:** Teste instável que passa ou falha sem mudanças aparentes de código.

- **Assert:** Comando que verifica se o resultado obtido é o que era esperado hoje.

- **Mock:** Simulação de um comportamento ou dado de um serviço externo no teste.

- **Page Object:** Padrão de design que abstrai a interface da lógica de teste real.

- **CI/CD Integration:** Execução automática de testes em cada deploy ou merge sim.

- **Bundle Size:** Tamanho total dos arquivos de código enviados ao navegador hoje.

- **Core Web Vitals:** Métricas de performance essenciais para a UX e SEO do Google.

- **User Story:** Descrição de uma funcionalidade do ponto de vista do usuário final.

- **Acceptance Criteria:** Condições que um software deve satisfazer para ser aceito.

- **Unit Test:** Teste da menor unidade de código isolada (ex: uma função pura) sim.

- **Integration Test:** Teste que valida a comunicação entre múltiplos módulos/serviços.

- **Black Box:** Teste focado na funcionalidade sem conhecimento interno do código.
