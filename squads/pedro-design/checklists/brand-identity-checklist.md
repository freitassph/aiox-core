# Brand Identity Checklist

Checklist completo de validação para identidade visual. Zero mediocridade tolerada.
Toda entrega de branding DEVE passar por cada item antes de ser considerada completa.

---

## 1. Brand Discovery & Strategy

### 1.1 Brand Platform
- [ ] **Propósito:** "Por que existimos além do lucro?" — claramente articulado
- [ ] **Visão:** Estado futuro em 10 anos definido com especificidade
- [ ] **Missão:** O que fazemos, para quem, como — resposta em 1 frase
- [ ] **Valores:** 3-5 valores com comportamentos concretos associados
- [ ] **Personalidade:** 5 adjetivos de marca validados pelo cliente
- [ ] **Brand Voice:** Tom definido em 3 dimensões (ex: Confiante, Acolhedor, Preciso)
- [ ] **Posicionamento:** Diferenciador único validado vs. concorrentes

### 1.2 Análise Competitiva
- [ ] Mínimo 5 concorrentes diretos mapeados visualmente
- [ ] 3 concorrentes indiretos incluídos
- [ ] Whitespace de diferenciação identificado
- [ ] Anti-referências documentadas (o que NÃO queremos parecer)
- [ ] Referências aprovadas documentadas com raciocínio

### 1.3 Audiência
- [ ] Persona primária definida (demografia, psicografia, comportamentos)
- [ ] Persona secundária definida (se aplicável)
- [ ] Pesquisa qualitativa realizada (mínimo 5 entrevistas OU survey com n>30)
- [ ] Jobs-to-be-done mapeados

---

## 2. Processo Visual

### 2.1 Moodboards
- [ ] 3 direções visuais distintas apresentadas
- [ ] Cada moodboard com 12-16 referências coerentes
- [ ] Direção escolhida documentada com justificativa
- [ ] Feedback do cliente registrado e incorporado

### 2.2 Conceitos de Logo
- [ ] Mínimo 3 conceitos desenvolvidos (não esboços — artes finalizadas)
- [ ] Cada conceito com variação horizontal + vertical
- [ ] Conceitos apresentados em contexto real (mockups)
- [ ] Conceito aprovado selecionado com feedback documentado

---

## 3. Sistema de Logo

### 3.1 Construção e Geometria
- [ ] Grid de construção documentado
- [ ] Proporções definidas em valores absolutos
- [ ] Clear space mínimo definido (recomendado: X = altura do ícone)
- [ ] Tamanho mínimo de uso: digital (32px) + impresso (20mm) definidos
- [ ] Alinhamento óptico corrigido (não apenas matemático)

### 3.2 Variações Obrigatórias
- [ ] **Primária:** Versão principal colorida (horizontal ou stacked)
- [ ] **Stacked/Vertical:** Se aplicável à proporção do logo
- [ ] **Monocromática:** Versão em preto puro (#000000)
- [ ] **Negativo/Branco:** Versão em branco puro (#FFFFFF)
- [ ] **Favicon/App Icon:** Versão simplificada para 16-32px
- [ ] **Símbolo isolado:** Se houver ícone separável do wordmark

### 3.3 Usos Incorretos (Logo Don'ts)
- [ ] Distorção proibida documentada
- [ ] Rotação proibida documentada
- [ ] Cores fora do sistema proibidas documentadas
- [ ] Efeitos proibidos (sombra, gradiente não-aprovado) documentados
- [ ] Fundo proibido documentado
- [ ] Mínimo 8 usos incorretos ilustrados

### 3.4 Teste de Resistência (Logo Test Matrix)
- [ ] ✅ Funciona em 16x16px (favicon)
- [ ] ✅ Funciona em 1200x630px (OG image)
- [ ] ✅ Funciona em preto e branco
- [ ] ✅ Funciona em negativo (fundo escuro)
- [ ] ✅ Funciona impresso em 1 cor
- [ ] ✅ Funciona bordado (sem degradês, mínimo de cores)
- [ ] ✅ Funciona em relevo/hot stamp
- [ ] ✅ Funciona em fundo fotográfico claro
- [ ] ✅ Funciona em fundo fotográfico escuro
- [ ] ✅ Legível a 5 metros de distância
- [ ] ✅ Reconhecível em 3 segundos por pessoa nova à marca
- [ ] ✅ Testado em contexto mobile (notificação, ícone de app)
- [ ] ✅ Testado em assinatura de email
- [ ] ✅ Testado em cartão de visita (tamanho real)

---

## 4. Sistema de Cores

### 4.1 Definição do Palette

#### Cores Primárias
- [ ] **HEX** definido (ex: #2563EB)
- [ ] **RGB** definido (ex: 37, 99, 235)
- [ ] **HSL** definido (ex: 221°, 83%, 53%)
- [ ] **CMYK** definido para impressão (ex: 84, 58, 0, 8)
- [ ] **Pantone** definido se aplicável (ex: Pantone 285 C)
- [ ] Ratio de contraste verificado vs. branco: __ :1 (min 4.5:1 para WCAG AA)
- [ ] Ratio de contraste verificado vs. preto: __ :1

#### Cores Secundárias
- [ ] Paleta secundária definida (2-4 cores)
- [ ] Uso de cada cor secundária documentado
- [ ] Todos os valores (HEX/RGB/CMYK/Pantone) definidos

#### Neutros
- [ ] Escala de neutros definida (mínimo 6 tons)
- [ ] Distinction entre neutros quentes e frios documentada

#### Semânticas
- [ ] **Sucesso/Verde:** HEX + WCAG ratio
- [ ] **Erro/Vermelho:** HEX + WCAG ratio
- [ ] **Aviso/Amarelo:** HEX + WCAG ratio
- [ ] **Info/Azul:** HEX + WCAG ratio

### 4.2 Regras de Uso
- [ ] Proporção de uso documentada (primária / secundária / neutros)
- [ ] Combinações aprovadas documentadas
- [ ] Combinações proibidas documentadas
- [ ] Dark mode palette definida (se aplicável)

### 4.3 Acessibilidade de Cor
- [ ] Texto normal (#body) vs. fundo principal: ≥ 4.5:1 WCAG AA ✓
- [ ] Texto grande (≥18pt ou ≥14pt bold) vs. fundo: ≥ 3:1 WCAG AA ✓
- [ ] Elementos UI (botões, inputs) vs. fundo: ≥ 3:1 ✓
- [ ] Funciona para deuteranopia (daltonismo vermelho-verde) — verificado com simulador ✓
- [ ] Funciona para protanopia (daltonismo vermelho) ✓
- [ ] Funciona para tritanopia (daltonismo azul-amarelo) ✓

---

## 5. Sistema Tipográfico

### 5.1 Seleção de Fontes
- [ ] **Display/Heading:** Família selecionada com justificativa estratégica
  - [ ] Licença verificada (web + digital + impressão)
  - [ ] Variações usadas documentadas (weights, styles)
  - [ ] Fonte de fallback definida
- [ ] **Body/Text:** Família selecionada
  - [ ] Licença verificada
  - [ ] Legibilidade testada em corpo 14px/16px
  - [ ] Fonte de fallback definida
- [ ] **Mono/Code:** Família selecionada (se aplicável)
- [ ] **Accent:** Família de destaque (se aplicável — uso esparso)

### 5.2 Escala Tipográfica
- [ ] **Display XL:** Tamanho + peso + line-height + letter-spacing definidos
- [ ] **H1:** Definido
- [ ] **H2:** Definido
- [ ] **H3:** Definido
- [ ] **H4:** Definido
- [ ] **Body Large:** Definido
- [ ] **Body Default:** Definido
- [ ] **Body Small:** Definido
- [ ] **Caption:** Definido
- [ ] **Label:** Definido
- [ ] **Overline:** Definido

### 5.3 Regras de Uso
- [ ] Hierarquia visual documentada
- [ ] Combinações de heading + body aprovadas
- [ ] Usos proibidos (ex: italic em serif small) documentados
- [ ] Uso em dark mode especificado (opacidade, peso ajustado)

---

## 6. Sistema de Iconografia

### 6.1 Definição
- [ ] Biblioteca de ícones selecionada (Lucide, Phosphor, Material, custom, etc.)
- [ ] Estilo definido: **stroke** (outline) ou **fill** — NÃO misturado
- [ ] Tamanhos padrão: 16px, 20px, 24px, 32px definidos
- [ ] Stroke weight padrão definido (ex: 1.5px, 2px)
- [ ] Corner radius dos ícones alinhado com a brand personality

### 6.2 Custom Icons (se aplicável)
- [ ] Grid de construção (ex: 24x24 com 2px margin = área 20x20)
- [ ] Estilo consistente em todos os ícones
- [ ] Todos os ícones testados em sizes 16px e 24px
- [ ] Arquivo master em SVG com camadas organizadas

---

## 7. Sistema de Imagens e Fotografia

### 7.1 Estilo Fotográfico
- [ ] Mood fotográfico definido (ex: "Autêntico, luz natural, pessoas reais")
- [ ] Paleta de cor fotográfica documentada (LUT ou preset nomeado)
- [ ] Exemplos "faça isso" e "não faça isso" documentados
- [ ] Diretrizes de seleção de modelo (diversidade, autenticidade)

### 7.2 Ilustrações e Gráficos
- [ ] Estilo de ilustração definido (se aplicável)
- [ ] Uso de imagens AI documentado (permitido/restrito/proibido)
- [ ] Estilos de gráfico de dados definidos (cores, tipografia)

---

## 8. Voz e Tom

### 8.1 Brand Voice
- [ ] 4 dimensões de voz documentadas com exemplos
- [ ] Exemplos "escrevemos assim" vs. "não escrevemos assim" para cada dimensão
- [ ] Vocabulário da marca (palavras que usamos, palavras que evitamos)
- [ ] Nome do produto/empresa: capitalização e formatação definidas

### 8.2 Tom por Contexto
- [ ] Tom para marketing/aquisição
- [ ] Tom para onboarding
- [ ] Tom para erros/problemas
- [ ] Tom para celebração/sucesso
- [ ] Tom para comunicações legais/compliance

---

## 9. Aplicações

### 9.1 Papelaria (se aplicável)
- [ ] Cartão de visita: frente + verso
- [ ] Papel timbrado
- [ ] Envelope
- [ ] Assinatura de email

### 9.2 Digital
- [ ] Favicon (16px, 32px, 180px Apple Touch Icon)
- [ ] OG Image template (1200x630px)
- [ ] Social media profiles (avatar + cover — tamanhos por plataforma)
- [ ] Loading screen / splash screen (se app mobile)

### 9.3 Ambientes (se aplicável)
- [ ] Sinalização/signage
- [ ] Uniforme/vestuário
- [ ] Embalagem

### 9.4 Mínimo de Aplicações Entregues
- [ ] ✅ Mínimo 10 aplicações criadas e aprovadas
- [ ] ✅ Cada aplicação em mockup fotorrealista
- [ ] ✅ Mix de contextos (digital + físico se aplicável)

## 10. Entregáveis Finais

### 10.1 Arquivos de Logo
- [ ] **SVG** master de todas as variações
- [ ] **PNG @1x** de todas as variações
- [ ] **PNG @2x** de todas as variações
- [ ] **PNG @4x** de todas as variações (para uso em retina/print)
- [ ] **EPS** para impressão (se necessário)
- [ ] **PDF** versão vetorial para impressão
- [ ] Nomenclatura de arquivos padronizada (ex: `logo-primary-color.svg`)
- [ ] Pasta organizada por variação

### 10.2 Brandbook
- [ ] Capítulo 1: Brand Story (Purpose, Vision, Values, Personality)
- [ ] Capítulo 2: Logo (construção, variações, usos corretos e incorretos)
- [ ] Capítulo 3: Color System (todos os valores, ratios de acessibilidade)
- [ ] Capítulo 4: Typography (famílias, escala, regras)
- [ ] Capítulo 5: Imagery Style (fotografia, ilustrações)
- [ ] Capítulo 6: Iconography (sistema, uso, estilo)
- [ ] Capítulo 7: Applications (10+ aplicações)
- [ ] Capítulo 8: Voice & Tone (adjetivos, exemplos, glossário)
- [ ] **PDF final exportado** (alta resolução + versão web)
- [ ] **Figma/Canva source** entregue ao cliente

### 10.3 Design Tokens Digitais
- [ ] `design-tokens.json` exportado com estrutura brand/color/font
- [ ] Tokens validados por desenvolvedor (importação funcional)
- [ ] CSS custom properties geradas
- [ ] Figma Variables configuradas (se Figma foi usado)

### 10.4 Handoff
- [ ] Briefing de uso entregue ao cliente (1 pager)
- [ ] Treinamento básico para equipe interna (se necessário)
- [ ] Onde guardar / como usar os arquivos documentado

---

## 11. Auditoria Final pelo @pdg-auditor

### 11.1 Consistência Interna
- [ ] Cores do logo = cores definidas no sistema de cor ✓
- [ ] Tipografia do logo (se wordmark) = família tipográfica do sistema ✓
- [ ] Personalidade do logo alinhada com brand personality articulada ✓
- [ ] Tom visual coerente entre todas as aplicações ✓

### 11.2 Diferenciação
- [ ] Logo claramente diferenciado dos 5 concorrentes diretos ✓
- [ ] Cor primária diferenciada do líder de categoria (exceto intencional) ✓
- [ ] Sem clichês visuais do setor não-intencionais ✓

### 11.3 Versatilidade
- [ ] Todos os 14 testes da Logo Test Matrix passaram ✓
- [ ] Funciona com budget de impressão limitado (1-2 cores) ✓
- [ ] Escalável de favicon a outdoor ✓

### 11.4 Qualidade Técnica
- [ ] SVG limpo (sem paths desnecessários, otimizado com SVGO) ✓
- [ ] Cores em modo correto (RGB para digital, CMYK para impressão) ✓
- [ ] Fontes convertidas para outlines nos arquivos de logo ✓
- [ ] Nenhum efeito raster em arquivos vetoriais ✓
