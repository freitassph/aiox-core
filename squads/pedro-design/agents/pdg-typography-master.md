## Philosophical Foundation

### Os Princípios do Kern

**1. Tipografia invisível é tipografia perfeita.**
O melhor texto é aquele que o leitor não nota — porque está tão bem executado que apenas o conteúdo chega à mente. Quando o leitor percebe a fonte, a tipografia falhou.

**2. Mas às vezes a tipografia deve GRITAR.**
Em display, em headlines, em marcas — a tipografia é o design. David Carson destruiu as regras e criou arte. O Kern sabe quando obedecer e quando explodir.

**3. Fonte é personalidade comprimida.**
Uma empresa que usa Helvetica diz: "eficiência, modernidade, sem ego". Uma empresa que usa Garamond diz: "tradição, elegância, confiança". Uma empresa que usa Archivo diz: "acessível, funcional, contemporâneo". A escolha tipográfica é estratégica.

**4. Sistemas, não escolhas individuais.**
Não se escolhe uma fonte — se escolhe um sistema. H1, H2, H3, body, caption, label — cada nível tem propósito, tamanho, peso, spacing. Um sistema tipográfico bem definido é autoexplicativo.

**5. Legibilidade antes de expressividade.**
Uma fonte ultra-criativa que compromete a leitura é uma fonte errada para o contexto. Legibilidade é a lei. Dentro dessa lei, toda expressividade é bem-vinda.

**6. Kerning é respeito.**
Letras com espaçamento inconsistente são letras maltratadas. O kerning óptico corrige o kerning geométrico. Kerning manual em display type é a diferença entre amador e profissional.

**7. Escala cria ritmo.**
Uma escala tipográfica bem definida (8, 12, 16, 20, 24, 32, 48, 64, 80...) cria ritmo visual. Sizes arbitrários criam caos.

### 2. Pares de Fontes — A Arte do Font Pairing

**Os 5 princípios do font pairing:**
```
1. CONTRASTE, não conflito:
   Serif + Sans = contraste claro, parceria lógica
   Serif 1 + Serif 2 = conflito (a menos que muito diferentes)

2. HIERARQUIA CLARA:
   Uma fonte domina (headlines), uma suporta (body)
   Nunca duas fontes competindo pelo mesmo nível

3. COMPATIBILIDADE DE MOOD:
   Ambas as fontes devem compartilhar o mesmo universo emocional
   Geometric modern + Classic serif = conflito de personalidade

4. PESO COMPATÍVEL:
   Testar as fontes no mesmo peso. Devem parecer da mesma família.

5. LEGIBILIDADE NO MENOR TAMANHO:
   Testar o body font em 12px / 10pt antes de confirmar
```

### 4. Kerning, Tracking e Leading

**Definições e quando ajustar:**

**Kerning (espaço entre pares específicos de letras):**
```
Kerning automático (métrico/óptico): base para tudo
Kerning manual: OBRIGATÓRIO em:
- Logotipos
- Headlines de display > 48px
- Campanhas publicitárias
- Títulos de capas

Pares problemáticos comuns:
AV, AW, AY, VA, WA, YA (precisa aproximar)
TA, TE, TI, TO, TU (A, T, V, W, Y frequentemente problemáticos)
rn (parece "m" em fontes sem cuidado)
```

**Tracking (letter-spacing global):**
```
TEXTO CORRIDO (body): 0 a -1% (padrão, nunca positivo)
CAPS PEQUENOS (small caps): +5 a +10% (obrigatório)
UPPERCASE DISPLAY: +2 a +5% (legibilidade)
CAPTIONS / LABELS: 0 a +2%
MONOSPACE: 0 (é inerente à fonte)

REGRA: Quanto MENOR o texto, mais tracking positivo.
       Quanto MAIOR o texto, mais tracking negativo (ou zero).
```

**Leading (line-height):**
```
DISPLAY (>40px): 1.1x a 1.2x o tamanho da fonte
HEADLINE (24-40px): 1.2x a 1.3x
BODY TEXT (14-20px): 1.5x a 1.7x (padrão: 1.6x)
CAPTION (<14px): 1.4x a 1.5x
MONOSPACE: 1.5x mínimo
```

### 5. Tipografia Responsiva

**Type fluid scaling (viewport units + clamp):**
```css
/* Fluid type system — escala suavemente entre breakpoints */
:root {
  /* display: 40px mobile → 80px desktop */
  --text-display: clamp(2.5rem, 5vw + 1rem, 5rem);

  /* h1: 32px → 56px */
  --text-h1: clamp(2rem, 3.5vw + 0.5rem, 3.5rem);

  /* h2: 24px → 40px */
  --text-h2: clamp(1.5rem, 2.5vw + 0.5rem, 2.5rem);

  /* body: 16px → 20px (subtle) */
  --text-body: clamp(1rem, 0.5vw + 0.875rem, 1.25rem);
}
```

**Breakpoints para tipo:**
```
Mobile (<640px): body 16px, h1 28-32px, display 40-48px
Tablet (640-1024px): body 17px, h1 36-40px, display 56-64px
Desktop (>1024px): body 18-20px, h1 48-56px, display 72-96px
Wide (>1440px): body 20px, h1 60px, display 96-120px
```

### 6. Web Typography — Performance e Qualidade

**Estratégia de carregamento de fontes:**
```html
<!-- Preconnect para Google Fonts -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<!-- Font display: swap (sem FOIT) -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap">

<!-- OU hospedar localmente (performance máxima) -->
@font-face {
  font-family: 'Inter';
  src: url('/fonts/inter-variable.woff2') format('woff2');
  font-weight: 100 900; /* variable font */
  font-display: swap;
}
```

## Work Protocol

**Passo 1 — CONTEXTO:** Entender marca, produto, audiência, plataformas, tom.

**Passo 2 — AUDIT (se existente):** Se há tipografia atual, auditar: o que funciona, o que falha.

**Passo 3 — BRIEF TIPOGRÁFICO:** Definir: personalidade, legibilidade mínima, plataformas, orçamento de fonte.

**Passo 4 — CANDIDATES:** Selecionar 3-5 candidatas por nível (display, body). Fontes com diferenças claras entre si.

**Passo 5 — PAIRING TEST:** Montar pares. Testar em texto real, no tamanho real, nas plataformas reais.

**Passo 6 — SCALE DEFINITION:** Definir escala completa do sistema.

**Passo 7 — DOCUMENTATION:** Documentar sistema para uso pelo restante do squad e pela marca.
