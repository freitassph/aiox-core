# Checklist: Client Delivery

## Como usar

Aplique este checklist em **toda e qualquer entrega ao cliente** — relatórios, criativos, campanhas, estratégias, apresentações, dados ou qualquer outro output.
O pa-anchor valida a completude antes de qualquer envio.

**Princípio fundamental:** O cliente não deve receber nada que precise ser refeito por erro nosso. Cada entrega é um reflexo do nosso padrão de excelência.

---

## 🔴 SEÇÃO 1: Quality Review — Entregue perfeito ou não entregue

### Revisão de Conteúdo
- [ ] O entregável responde completamente ao brief ou solicitação original
- [ ] Todos os pontos do brief foram endereçados (checagem item por item)
- [ ] Não há informações contraditórias dentro do documento/material
- [ ] Versão correta do arquivo está sendo enviada (não rascunho ou versão anterior)
- [ ] Nenhum placeholder, "TBD" ou texto de rascunho remanescente
- [ ] Dados e números conferidos com a fonte original
- [ ] Comparações com período anterior verificadas
- [ ] Metas e targets do cliente confirmados antes da análise de performance

### Revisão de Forma
- [ ] Ortografia e gramática revisadas (ferramenta + humano)
- [ ] Formatação consistente ao longo de todo o documento
- [ ] Identidade visual do cliente aplicada corretamente (se template do cliente)
- [ ] Logo do cliente na versão e posição corretas
- [ ] Template da agência aplicado corretamente (se template próprio)
- [ ] Fontes incorporadas ou sistema safe fonts usadas
- [ ] Imagens em alta resolução e sem pixelização
- [ ] Numeração de páginas correta
- [ ] Sumário/índice atualizado (para documentos longos)
- [ ] Cabeçalhos e rodapés corretos com nome do cliente e data

### Revisão Técnica
- [ ] Arquivo abre corretamente em pelo menos 2 aplicativos/sistemas diferentes
- [ ] Links internos ao documento funcionam
- [ ] Links externos mencionados estão funcionais
- [ ] Gráficos e tabelas renderizam corretamente
- [ ] Fórmulas e cálculos em spreadsheets verificados
- [ ] Arquivo não está corrompido (verificado após compactação)
- [ ] Tamanho do arquivo adequado para envio por e-mail (< 25MB) ou via link

---

## 🔴 SEÇÃO 2: Client Communication — Clareza acima de tudo

### Preparação da Comunicação
- [ ] Destinatários corretos identificados (cliente primário + cópias necessárias)
- [ ] Contato correto do cliente confirmado (e-mail/WhatsApp/plataforma)
- [ ] Contexto de entrega preparado (o que está sendo entregue e por quê)
- [ ] Próximos passos claros definidos na comunicação de entrega
- [ ] Prazo para feedback ou aprovação do cliente comunicado
- [ ] Destaques da entrega resumidos em 2-3 pontos para facilitar revisão

### Tom e Profissionalismo
- [ ] Linguagem formal e profissional mantida
- [ ] Saudação e fechamento adequados ao nível de relacionamento
- [ ] Nenhuma informação confidencial de outros clientes mencionada
- [ ] Expectativas realistas comunicadas (sem overpromise)
- [ ] Eventuais pontos de atenção ou limitações comunicados proativamente

### Timing
- [ ] Entrega dentro do prazo combinado
- [ ] Se atrasado: cliente foi informado previamente com novo prazo
- [ ] Entrega em horário comercial (evitar envios fora do horário ou finais de semana, salvo urgência)
- [ ] Entrega não conflita com eventos do cliente (lançamento, reunião de board, etc.)

---

## 🔴 SEÇÃO 3: Files & Assets — Tudo organizado e acessível

### Organização de Arquivos
- [ ] Nomenclatura de arquivo clara e padronizada (Cliente_TipodeEntrega_AAAAMM_vFinal)
- [ ] Versão final claramente identificada (sem ambiguidade com versões anteriores)
- [ ] Arquivos agrupados logicamente (por tipo, por campanha, por período)
- [ ] Pasta compartilhada com estrutura clara e nomeação consistente
- [ ] Arquivos de suporte incluídos (fontes, imagens em alta, assets editáveis)

### Formatos de Entrega
- [ ] Formato correto para o uso do cliente (PDF para leitura, PPT para edição, etc.)
- [ ] Resolução adequada para o uso pretendido (tela vs. impressão)
- [ ] Versão editável incluída quando necessário (não só PDF)
- [ ] Arquivos de fonte (PSD, AI, Figma) entregues quando contratado
- [ ] Vídeos em formato compatível com os players do cliente

### Acesso & Segurança
- [ ] Link de compartilhamento com permissões corretas (visualizar vs. editar)
- [ ] Password de acesso comunicada em canal separado (se aplicável)
- [ ] Link com prazo de validade adequado (não expirar antes do cliente usar)
- [ ] Backup dos arquivos feito no servidor da agência antes de enviar
- [ ] Informações sensíveis protegidas adequadamente

---

## 🔴 SEÇÃO 4: Documentation — O contexto que salva vidas

### Contexto da Entrega
- [ ] Resumo executivo presente (especialmente para documentos longos)
- [ ] Metodologia explicada quando não for óbvia
- [ ] Glossário de termos técnicos incluído (se entrega para não-especialistas)
- [ ] Fonte dos dados especificada (plataforma + período de extração)
- [ ] Limitações e ressalvas documentadas (dados parciais, amostras pequenas, etc.)
- [ ] Comparativos com benchmarks do mercado incluídos quando relevante

### Histórico e Rastreabilidade
- [ ] Versão e data da entrega registradas no documento
- [ ] Histórico de revisões documentado (quem pediu o quê, quando)
- [ ] Nome do responsável pela entrega registrado
- [ ] Correspondência de referência mencionada quando aplicável
- [ ] Número da story ou projeto referenciado (para rastreamento interno)

### Handoff Técnico (quando aplicável)
- [ ] Documentação técnica incluída (para entregas de desenvolvimento/tech)
- [ ] Instruções de uso ou implementação presentes
- [ ] Credenciais entregues em meio seguro (nunca por e-mail em texto puro)
- [ ] Dependências e pré-requisitos listados
- [ ] Contato de suporte disponível comunicado

---

## 🟡 SEÇÃO 5: Approval Process — Fechando o loop formal

### Processo de Aprovação
- [ ] Deixar claro o que precisa de aprovação vs. o que é informativo
- [ ] Prazo de aprovação comunicado (ex.: "precisamos de retorno até X para manter o prazo Y")
- [ ] Alternativas apresentadas quando há decisão pendente
- [ ] Responsável pela aprovação no lado do cliente identificado
- [ ] Processo de aprovação alinhado previamente (e-mail, assinatura digital, reunião)

### Revisões & Feedback
- [ ] Número de rodadas de revisão incluídas no escopo comunicado ao cliente
- [ ] Processo para envio de feedback explicado
- [ ] Template ou guia de feedback disponibilizado (se entrega complexa)
- [ ] Prazo máximo para solicitação de revisões comunicado
- [ ] Status de revisões anteriores verificado (pendências de rounds anteriores)

### Formalização
- [ ] Aprovação por e-mail ou documento assinado solicitada
- [ ] Aprovação anterior de conceito verificada antes de entrar em produção final
- [ ] Ata de reunião de alinhamento disponível se houver dúvidas sobre escopo

---

## 🟡 SEÇÃO 6: Follow-up — O pós-entrega que fideliza

### Imediato (D+1 até D+3)
- [ ] Check-in com cliente confirmando recebimento da entrega
- [ ] Abertura para dúvidas sobre a entrega comunicada
- [ ] Qualquer pendência da entrega comunicada com prazo de resolução
- [ ] Satisfação inicial verificada (brief review call agendada se necessário)

### Curto Prazo (até 1 semana)
- [ ] Feedback formal do cliente solicitado
- [ ] Próximos passos e timeline do projeto atualizados
- [ ] Aprendizados da entrega documentados internamente
- [ ] Story/task marcada como concluída no sistema de gestão
- [ ] Nota NPS ou satisfação registrada internamente

### Relacionamento
- [ ] Entrega usada como oportunidade para reforçar valor entregue
- [ ] Upsell ou próxima fase identificada e registrada (sem pressão imediata)
- [ ] Case de sucesso documentado para portfólio (com autorização do cliente)
- [ ] Testemunho solicitado quando resultado for excepcional

---

## Categorias de Entrega — Especificações

### Relatório (Semanal / Mensal / Trimestral)
- [ ] Período de análise claramente especificado
- [ ] Acesso às plataformas verificado antes da extração de dados
- [ ] Dados extraídos no mesmo período e fuso horário
- [ ] Gráficos com eixos claramente nomeados
- [ ] Análise qualitativa além dos números brutos
- [ ] Recomendações acionáveis presentes

### Criativos & Assets
- [ ] Todos os formatos solicitados entregues
- [ ] Revisão de arquivo editável incluída
- [ ] Guia de uso dos assets incluído
- [ ] Diretrizes de aplicação comunicadas

### Estratégia & Planejamento
- [ ] Fundamentação em dados de mercado/pesquisa
- [ ] Plano de ação com responsáveis e prazos
- [ ] KPIs de sucesso definidos
- [ ] Premissas documentadas
