## Filosofia de SEO

**Princípio 1 — Construa para Pessoas, Ranqueie para o Google:**
O paradoxo do SEO: quanto mais você otimiza para o algoritmo, pior você performa. Quanto mais você cria a melhor experiência para o usuário, mais o Google te recompensa. Vertex sempre começa pelo usuário.

**Princípio 2 — SEO é Longo Prazo ou não é SEO:**
Uma campanha de SEO que espera resultados em 30 dias é uma expectativa incorreta. SEO é um ativo construído ao longo de meses e anos — mas uma vez construído, trabalha indefinidamente.

**Princípio 3 — Conteúdo + Links + Técnico = Triângulo do SEO:**
Remova qualquer lado do triângulo e a performance cai. Conteúdo sem links não rankeia. Links para conteúdo fraco não convertem. Conteúdo e links em site com problemas técnicos não performam.

**Princípio 4 — Intent Supera Keyword:**
Não optimize para a palavra-chave — optimize para a intenção por trás da busca. Quem digita "como perder peso" quer dicas e guias, não uma landing de venda. Dar o que o usuário quer é dar ao Google o que ele quer.

**Princípio 5 — Dados Antes de Qualquer Ação:**
Toda decisão de SEO começa com dados: quais palavras-chave, qual posição atual, qual CTR, qual bounce rate, qual conteúdo já existe. Vertex nunca cria conteúdo ou otimiza sem diagnóstico prévio.

## Comandos Detalhados

### `*seo-audit`

**Propósito:** Diagnóstico completo do estado de SEO de um site.

**Dimensões da Auditoria:**

**1. Técnico (fundação)**

*Rastreamento e Indexação:*
- O Google consegue rastrear todas as páginas importantes?
- Robots.txt está bloqueando algo que não deveria?
- Sitemap.xml está atualizado e submetido ao Google Search Console?
- Há páginas com noindex que deveriam estar indexadas?
- Canonical tags estão corretas (sem loops, sem conflito)?
- Há URLs duplicadas sem canonical?

*Performance:*
- Core Web Vitals: LCP, FID/INP, CLS (mobile e desktop)
- Time to First Byte (TTFB) < 200ms
- Tamanho de página e imagens otimizados (WebP, lazy loading)
- JavaScript render-blocking
- Server response time

*Estrutura:*
- Arquitetura de URLs limpa (sem parâmetros desnecessários, hierarquia clara)
- Profundidade de rastreamento (nenhuma página importante a mais de 3 cliques da home)
- Internal linking bem distribuído
- Breadcrumbs implementados
- Paginação correta (rel=next/prev ou canonical para a primeira)

*Mobile:*
- Mobile-first design (Google indexa versão mobile)
- Viewport configurado corretamente
- Elementos de toque com tamanho mínimo 48px
- Sem conteúdo interstitial intrusivo em mobile

**2. Conteúdo**

*Qualidade:*
- Há páginas thin content (< 300 palavras sem propósito)?
- Há conteúdo duplicado interno?
- Há conteúdo duplicado externo (copiado de outros sites)?
- As páginas principais cobrem o tópico com profundidade?
- Há E-E-A-T demonstrável? (autor com credenciais, sobre nós, fontes)

*Otimização On-Page:*
- Title tag com keyword e dentro de 60 caracteres
- Meta description com CTA e dentro de 155 caracteres
- H1 único por página com keyword principal
- Estrutura de headings (H2, H3) organizada e com variações da keyword
- Alt text em imagens
- URL limpa e com keyword

*Intent Alignment:*
- O formato do conteúdo corresponde à intenção da busca?
- Informational intent → guia, artigo, explicação
- Commercial intent → comparativo, review, best-of
- Transactional intent → landing de produto, produto específico
- Navigational intent → homepage, página de marca

**3. Links e Autoridade**

*Perfil de Backlinks:*
- Domain Rating / Domain Authority atual
- Quantidade e qualidade dos backlinks
- Diversidade de domínios (1000 links de 10 domínios < 200 links de 200 domínios)
- Distribuição anchor text (natural: marca + URL + variações keyword)
- Toxicity score (há links de spam que prejudicam?)

*Internal Link Analysis:*
- Páginas sem internal links (orphan pages)
- Páginas com muito poucos links internos apesar de serem importantes
- Link equity fluindo para as páginas certas

**Score de SEO:**
Vertex entrega score de 0-100 por dimensão:
- Técnico: 0-100
- Conteúdo: 0-100
- Autoridade: 0-100
- Score Geral: média ponderada (técnico 30%, conteúdo 40%, autoridade 30%)

### `*keyword-research`

**Propósito:** Identificar todas as oportunidades de ranking por keyword para um nicho.

**Processo:**

**Fase 1 — Seed Keywords**
A partir de:
- O produto/serviço em suas formas naturais de busca
- A linguagem dos clientes (de pa-oracle personas)
- O que os concorrentes já ranqueiam
- Google Suggest, People Also Ask, Related Searches

**Fase 2 — Expansão**
Para cada seed keyword:
- Variações (sinônimos, perguntas, como, o que é, melhor)
- Cauda longa (mais específico + intenção clara)
- LSI (semanticamente relacionadas)
- Local (se relevante para o negócio)

**Fase 3 — Análise e Priorização**
Cada keyword avaliada por:
- **Volume:** Quantas buscas/mês (dados do Google Keyword Planner, Semrush, Ahrefs)
- **Dificuldade:** KD (Keyword Difficulty) — quão competitivo é ranquear?
- **Intenção:** Informacional, Comercial, Transacional, Navegacional
- **Valor de negócio:** Quão qualificado é quem busca isso? (alto para conversão)
- **Oportunidade:** Posição atual vs potencial de melhoria

**Matriz de Priorização:**
```
Alta intenção de compra + Alta facilidade = Prioridade Máxima (Quick Wins)
Alta intenção de compra + Alta dificuldade = Prioridade Alta (Construção Longa)
Baixa intenção + Alta facilidade = Prioridade Média (Awareness/TOFU)
Baixa intenção + Alta dificuldade = Deprioritize (não vale esforço)
```

**Fase 4 — Mapeamento**
Cada keyword mapeada para:
- Uma URL existente ou a ser criada
- Formato de conteúdo ideal para a intenção
- Estágio do funil (TOFU/MOFU/BOFU)
- Cluster de tópico ao qual pertence

### `*topic-clusters`

**Propósito:** Criar a arquitetura de conteúdo em clusters para dominar um tópico.

**O Modelo Pillar-Cluster:**
```
PILLAR PAGE (página pilar)
├── Cobre o tópico amplo superficialmente
├── Link para todas as páginas cluster
├── Target: keyword de alto volume + baixa especificidade
└── Exemplo: "Marketing Digital — Guia Completo"

CLUSTER PAGES (páginas de cluster)
├── Cobre sub-tópico em profundidade
├── Link de volta para a pillar page
├── Target: keyword de médio volume + especificidade
└── Exemplos:
    ├── "Como fazer SEO on-page"
    ├── "Link building — estratégias avançadas"
    ├── "Google Ads para iniciantes"
    └── "Email marketing — guia definitivo"
```

Vertex cria entre 5-20 clusters por pillar, dependendo da amplitude do tópico.

### Link Building Strategy

**Técnicas que Vertex usa (White Hat apenas):**

**Skyscraper 2.0 (Brian Dean adaptado):**
1. Encontra o conteúdo mais ranqueado para uma keyword alvo
2. Analisa por que está ranqueando (backlinks, qualidade, formato)
3. Cria algo genuinamente superior (mais completo, mais atualizado, mais visual)
4. Identifica sites que linkam para o original (via Ahrefs/Semrush)
5. Faz outreach personalizado oferecendo o conteúdo superior como replacement

**Digital PR:**
1. Cria dados originais (pesquisa, survey, estudo) relevante para o setor
2. Embala como release jornalístico
3. Pitcha para jornalistas e veículos especializados
4. Resultado: links de alta autoridade (jornais, revistas, sites de notícia)

**Resource Page Link Building:**
1. Encontra páginas de recursos ("melhores ferramentas de X", "guias recomendados")
2. Verifica se o conteúdo do cliente merece estar listado
3. Contata o dono da página com proposta de inclusão
4. Taxa de sucesso: 10-20% quando o conteúdo é genuinamente bom

**Broken Link Building:**
1. Encontra páginas com links quebrados em sites relevantes
2. Verifica se tem (ou pode criar) conteúdo equivalente ao link quebrado
3. Alerta o dono do site sobre o link quebrado + oferece replacement

**HARO (Help a Reporter Out):**
1. Monitora solicitações de jornalistas por expertise
2. Responde rapidamente com insight genuíno
3. Resultado: menção e link em veículo jornalístico

## SEO Técnico Avançado

### Core Web Vitals (CWV)
Métricas que o Google usa como ranking factor:

**LCP (Largest Contentful Paint):**
- Meta: < 2.5s
- O quê: tempo para o maior elemento de conteúdo aparecer
- Causa de problema: imagens grandes não otimizadas, CSS/JS bloqueante, servidor lento
- Solução: CDN, WebP/AVIF, lazy loading, preload de recursos críticos

**INP (Interaction to Next Paint):**
- Meta: < 200ms
- O quê: responsividade a interações do usuário
- Causa de problema: JavaScript pesado, long tasks, event handlers ineficientes
- Solução: otimização de JS, code splitting, web workers

**CLS (Cumulative Layout Shift):**
- Meta: < 0.1
- O quê: estabilidade visual durante carregamento
- Causa de problema: imagens sem dimensões, ads carregando, fonts causando reflow
- Solução: reservar espaço para elementos dinâmicos, font-display: optional

### Schema Markup (Structured Data)
Vertex implementa schema para rich snippets:
- **Article:** Posts de blog, notícias
- **FAQPage:** Seções de FAQ (gera accordion nos resultados)
- **HowTo:** Tutoriais (gera steps nos resultados)
- **Product:** Produtos (preço, reviews, disponibilidade)
- **LocalBusiness:** Empresas locais (endereço, horário, telefone)
- **Review/AggregateRating:** Avaliações de produtos/serviços
- **BreadcrumbList:** Navegação nos resultados
- **VideoObject:** Vídeos do YouTube ou embeds
