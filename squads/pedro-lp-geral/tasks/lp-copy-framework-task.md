# Task: Copy Framework — World-Class Conversion Copy

## Objetivo
Produzir toda a copy da landing page usando o sistema Ogilvy × Schwartz × Hormozi. Copy que converte, não copy que "soa bem".

## Agente Responsável
`lp-world-class-copywriter` (Hugo Ogilvy)

## Inputs Necessários
- `briefing-document.md`
- `brand-archetype-report.md`
- `avatar-profile.md`
- `awareness-strategy.md`

## Processo

### FASE 1 — Diagnóstico de Copy (ANTES de escrever)
```
1. Identificar Awareness Level do avatar (Schwartz)
2. Mapear: dor profunda → desejo profundo → transformação desejada
3. Identificar o Unique Mechanism (nome próprio se não tiver)
4. Listar Top 5 objeções a responder
5. Validar: existe Starving Crowd para esta oferta?
```

### FASE 2 — Hero Section
```
1. Gerar 10 headlines usando os 5 frameworks (escolher o melhor)
2. Escrever subheadline com estrutura: dor confirmada → possibilidade → mecanismo
3. Escrever body do hero (2-4 parágrafos de conexão)
4. Criar 5 variações de CTA para teste A/B
5. Validar: hero responde "O que é? Para quem? Por que agora?" em < 5s
```

### FASE 3 — Problem Section
```
1. Agitar a dor com especificidade cirúrgica (não genericamente)
2. Normalizar: "você não está sozinho"
3. Ampliar consequência: "se nada mudar..."
4. Pivot para esperança: "mas existe uma forma diferente"
```

### FASE 4 — Solution Section
```
1. Apresentar o Unique Mechanism com nome próprio
2. Explicar em 3 passos simples
3. Diferenciação: por que outros métodos falham?
4. Prova rápida do mecanismo (dado, estudo, caso)
```

### FASE 5 — Proof Section
```
1. Redigir ou reformular 3-6 depoimentos para máximo impacto
2. Criar social proof numerics: "X clientes já transformaram..."
3. Estruturar casos de sucesso: antes → depois → resultado em 1 parágrafo
```

### FASE 6 — Value Stack & Offer
```
1. Listar todos os componentes da oferta com valor percebido
2. Montar Value Stack (Hormozi): itens + valores individuais + total
3. Criar contraste preço real vs. valor percebido
4. Escrever copy da garantia (elimina o risco)
```

### FASE 7 — Objections & FAQ
```
1. Responder as 5 maiores objeções (estrutura: reconhece → reframe → prova)
2. Criar FAQ com 6-10 perguntas reais (pergunta + resposta + CTA)
```

### FASE 8 — Closing & CTA Final
```
1. Resumo bullet da oferta
2. Value stack condensado
3. Urgência real (com justificativa)
4. CTA mais específico e irresistível de toda a página
5. Anti-CTA: o que acontece se não agir?
```

## Output Esperado

- `hero-copy.md` — Headline, subheadline, body, CTA (5 variações)
- `sections-copy.md` — Copy completa de todas as seções
- `faq-copy.md` — FAQ com 6-10 perguntas e respostas
- `cta-variants.md` — Banco de CTAs para A/B testing
- `headline-bank.md` — 20+ headlines alternativas
- `meta-copy.md` — Title tag, meta description, OG tags
- `awareness-diagnosis.md` — Nível de consciência e estratégia aplicada
